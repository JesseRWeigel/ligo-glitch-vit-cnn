# Requirements: Transformer-Based Rare Glitch Classification for LIGO CW Searches

**Defined:** 2026-03-16
**Core Research Question:** Can a Vision Transformer trained on LIGO O3 spectrograms outperform Gravity Spy's CNN at classifying rare glitch morphologies, and can this improvement translate to better data quality for continuous gravitational wave searches?

## Primary Requirements

### Data Pipeline

- [ ] **DATA-01**: Download and validate O3 Gravity Spy labeled dataset from Zenodo (record 5649212), including pre-made spectrogram images and metadata
- [ ] **DATA-02**: Generate Q-transform spectrograms from O3 GWOSC strain data (H1 + L1) with controlled normalization, using GWpy at 4 kHz sample rate with 4 standard time windows
- [ ] **DATA-03**: Implement temporal train/val/test split (not random) to prevent data leakage from temporally correlated glitches; document split boundaries
- [ ] **DATA-04**: Audit rare-class label quality and document per-class sample counts across all Gravity Spy classes; flag classes with fewer than 25 labeled examples

### Model Architecture & Training

- [ ] **MODL-01**: Reproduce Gravity Spy CNN baseline with modern training recipe (AdamW optimizer, cosine learning rate schedule, identical augmentation to ViT) to establish a fair comparison floor
- [ ] **MODL-02**: Train ViT-B/16 fine-tuned from ImageNet-21k pretrained weights on Gravity Spy spectrograms with mixed precision (fp16) on RTX 5090
- [ ] **MODL-03**: Implement focal loss with class-balanced batch sampling to optimize for rare-class F1/recall

### Evaluation

- [ ] **EVAL-01**: Compute per-class F1 score and recall for both ViT and CNN on held-out O3 temporal test set, with emphasis on rare classes (classes with fewer than N_rare training examples)
- [ ] **EVAL-02**: Validate ViT trained on O3 against O4 public data to test distribution shift robustness; rare-class F1 must not degrade by more than 20% relative to O3 test performance
- [ ] **EVAL-03**: Statistical significance testing via bootstrap or permutation test: rare-class macro-averaged F1 improvement over CNN baseline must be significant at p < 0.05

### CW Sensitivity Study

- [ ] **CWSS-01**: Quantify CW search data quality improvement (duty cycle or strain upper limit) using ViT-based glitch vetoes vs CNN-based vetoes on the same O3 data segment

### Deliverables

- [ ] **DELV-01**: Trained ViT model with inference pipeline: model weights, preprocessing script (spectrogram generation), and inference script for classifying new glitches
- [ ] **DELV-02**: Research paper with method description, experimental results (per-class F1 tables, confusion matrices), CW sensitivity analysis, and comparison with Gravity Spy CNN

## Follow-up Requirements

### Extended Model Architecture

- **EXTD-01**: Multi-view attention fusion across 4 time-window spectrograms (following Wu et al. 2024 approach)
- **EXTD-02**: cDVGAN-based synthetic data augmentation for rare glitch classes
- **EXTD-03**: Audio Spectrogram Transformer (AST) comparison with AudioSet pretraining
- **EXTD-04**: Self-supervised pretraining (MAE or contrastive) on unlabeled Gravity Spy spectrograms

### Extended Analysis

- **EXTD-05**: ViT attention map analysis for interpretable glitch morphology features
- **EXTD-06**: Open-set recognition for novel O4 glitch morphologies not in O3 taxonomy

## Out of Scope

| Topic | Reason |
| ----- | ------ |
| Time-series transformers on raw strain | Project scoped to ViT on spectrograms; time-series approach is a separate study |
| Real-time / low-latency inference pipeline | Research focus, not deployment; latency optimization is engineering work |
| Full CW search pipeline (Viterbi, Sideband) | Only quantify data quality impact, not run full CW searches |
| Non-LIGO detectors (Virgo, KAGRA) | Different noise characteristics; generalization study is follow-up work |
| CBC detection or parameter estimation | Different signal type and search methodology |
| Quantum phase transitions / condensed matter | Not applicable to this ML/GW project |

## Accuracy and Validation Criteria

| Requirement | Accuracy Target | Validation Method |
| ----------- | --------------- | ----------------- |
| MODL-01 | Reproduce published Gravity Spy CNN metrics within 2% | Compare against Zevin et al. 2017 reported accuracy |
| MODL-02 | ViT rare-class macro-F1 > CNN rare-class macro-F1 | Per-class F1 comparison on temporal test set |
| EVAL-01 | Per-class F1 reported to 3 significant figures | Bootstrap 95% confidence intervals |
| EVAL-02 | O4 degradation < 20% relative to O3 | Direct comparison on O4 validation set |
| EVAL-03 | p < 0.05 for rare-class improvement | Bootstrap or permutation test with 10,000 resamples |
| CWSS-01 | Measurable duty cycle or upper limit improvement | Compare veto lists on identical data segment |
