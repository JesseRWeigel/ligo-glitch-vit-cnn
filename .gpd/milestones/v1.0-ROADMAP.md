# Roadmap: Transformer-Based Rare Glitch Classification for LIGO CW Searches

## Overview

This project investigates how Vision Transformers compare to CNNs for classifying LIGO glitch spectrograms, with focus on rare morphologies. The central finding is a **sample-efficiency threshold**: ViTs outperform CNNs for classes with ~100+ training samples but underperform for ultra-rare classes (<50 samples), revealing when the transformer's flexibility becomes a liability. The roadmap proceeds from data pipeline construction through fair baseline establishment, ViT training and per-class comparison, O4 distribution-shift validation of the threshold finding, CW sensitivity quantification, and paper delivery framing the threshold as the primary contribution.

### Phase 4: O4 Validation & CW Sensitivity

**Goal:** Both O3-trained models (ViT and CNN) are evaluated on O4 data to test whether the sample-efficiency threshold holds under distribution shift, and CW search data quality improvement from ViT-based glitch vetoes is quantified for the classes where ViT outperforms CNN.

**Depends on:** Phase 3

**Requirements:** EVAL-02, CWSS-01

**Contract Coverage:**
- Advances: claim-threshold (does the ~100-sample threshold hold on O4?), claim-cw-benefit (CW sensitivity quantification for ViT-advantaged classes)
- Deliverables: deliv-cw-analysis (CW sensitivity or duty cycle comparison plot with veto efficiency metrics)
- Anchor coverage: ref-gravity-spy -- CNN-based vetoes serve as the CW baseline comparison; O3 per-class performance serves as reference for O4 degradation
- Forbidden proxies: fp-qualitative-only -- CW benefit must be quantified as a measurable metric, not argued qualitatively
- Acceptance tests: test-threshold-o4 (does per-class ViT advantage correlate with training set size on O4?), test-o4-generalization (per-class F1 degradation analysis for both models), test-cw-improvement (measurable CW data quality improvement)

**Success Criteria** (what must be TRUE):

1. Both ViT and CNN evaluated on O4 public data; per-class F1 comparison on O4 mirrors the sample-efficiency threshold from O3 (ViT advantage for classes with 100+ train samples persists)
2. Per-class F1 degradation from O3->O4 documented for both models; overall degradation < 20% for both
3. CW-critical glitch classes identified (Scattered_Light, Violin_Mode, Low_Frequency_Lines, etc.); overlap with ViT-advantaged classes characterized
4. CW search data quality metric (duty cycle or strain upper limit) computed using ViT-based vetoes vs CNN-based vetoes on the same data segment; measurable improvement for ViT-advantaged CW-critical classes
5. deliv-cw-analysis produced with sensitivity comparison plot and veto efficiency metrics

**Backtracking trigger:** If O4 distribution shift destroys the threshold pattern (ViT loses advantage even for 100+ sample classes), the threshold finding may be O3-specific. Report with caveat. If CW improvement is not measurable, narrow to specific line-like glitch classes.

**Outcome (2026-03-17):**

1. ✓ Both models evaluated on 38,587 O4a glitches; CNN macro-F1=0.6674, ViT macro-F1=0.6695
2. ✓ Both pass <20% degradation threshold (CNN: −1.7%, ViT: −7.4%)
3. ✗ Sample-efficiency threshold NOT confirmed: Spearman ρ=−0.034, p=0.879; sign test 9/20 (chance)
4. ✗ O3 cross-check also non-significant (ρ=−0.119, p=0.59) — "threshold" was class-specific, not monotonic
5. ≈ CW veto at matched deadtime (22.4%): ViT efficiency 0.745 vs CNN 0.735 (~equal)
6. ✓ Power_Line: ViT F1 0.725 vs CNN 0.331 (+0.394) — strongest CW-relevant ViT advantage
7. **Second backtracking: paper must shift from "threshold" to "class-morphology-dependent architecture preferences"**
