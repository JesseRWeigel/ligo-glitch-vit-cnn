  <action>
    Create `scripts/18_power_analysis.py` that computes, for each rare class, the statistical power to detect a given F1 difference and the minimum detectable effect size (MDE).

    The question: "Given only n_test samples of class X in the test set, what is the smallest F1 difference between ViT and CNN we could reliably detect?"

    Target rare classes and their test-set sizes:
    - Chirp: n_test = 7
    - Wandering_Line: n_test = 6
    - 1080Lines: n_test = 6
    - Helix: n_test = 14
    - Light_Modulation: n_test = 66
    - Air_Compressor: n_test = 47
    - 1400Ripples: n_test = 33
    - Power_Line: n_test = 56

    Method -- simulation-based power analysis (more appropriate than parametric formulas for F1 on tiny samples):

    a) For each rare class with known n_test:
       - Define a grid of true effect sizes delta_F1 from 0.05 to 0.50 in steps of 0.05
       - For each delta_F1, simulate 10,000 iterations:
         * Sample CNN predictions: for each of n_test samples, predict correctly with probability p_cnn (use the observed per-class CNN recall as the baseline; if recall is 0, use 0.3 as a reasonable assumed baseline)
         * Sample ViT predictions: predict correctly with probability p_cnn - delta_F1 (or + delta_F1 for improvement direction)
         * Compute per-class F1 for each model from the simulated binary predictions (TP/FP/FN)
         * Apply a paired permutation test (or McNemar's test) at alpha=0.05 to determine if the difference is significant
       - Power = fraction of 10,000 iterations where the test rejects H0

    b) Compute MDE (minimum detectable effect) as the smallest delta_F1 where power >= 0.80.

    c) Compute power at the observed effect: use the actual observed macro-F1 difference of -0.062 as the effect size. Report power for each rare class at this specific effect.

    d) Also compute macro-F1 power analysis for the AGGREGATE rare-class metric:
       - Rare classes defined as train count < 200 (matching `configs/cnn_baseline.yaml` `evaluation.rare_threshold_train: 200`)
       - Simulate the rare-class macro-F1 (average of per-class F1 across all rare classes) under various effect sizes
       - Report MDE and power at observed aggregate effect (-0.062)

    e) Save to `results/06-computation-statistical-analysis/power_analysis.json`:
       - `per_class`: dict mapping class name to:
         * `n_test`: int
         * `observed_cnn_recall`: float (from Phase 2 results or hardcoded from STATE.md)
         * `minimum_detectable_effect`: float (MDE at power=0.80)
         * `power_at_observed_effect`: float (power at delta=-0.062)
         * `power_curve`: list of {delta, power} dicts
       - `aggregate_rare`:
         * `n_classes`: int
         * `mde`: float
         * `power_at_observed`: float
       - `methodology`: "simulation-based, 10K iterations, alpha=0.05, paired permutation test"
       - `conclusion`: auto-generated string: "underpowered" if power_at_observed < 0.80 for majority of rare classes, "adequately powered" otherwise

    f) Create `results/06-computation-statistical-analysis/ablation_summary.txt` combining both Task 1 and Task 2 results in a human-readable format suitable for Phase 7 paper writers:
       - Section 1: Random-split ablation result (accuracy, gap, comparison to Zevin et al.)
       - Section 2: Power analysis table (class, n_test, MDE, power_at_observed)
       - Section 3: Interpretation sentence for each finding

Fallback: compute from the existing test predictions by loading the CNN checkpoint and running inference on the temporal-split test set (the checkpoint and evaluation code already exist).
