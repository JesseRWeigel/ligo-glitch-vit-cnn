    **Step 1: Temporal split construction**

    Write `scripts/04_temporal_split.py` to:
    1. Load `data/metadata/gravity_spy_o3_filtered.csv` (output of Plan 01)
    2. Sort all samples by GPS event_time (ascending)
    3. Define split boundaries targeting ~70/15/15 ratio:
       - Compute the GPS time at the 70th percentile and 85th percentile of sorted event times
       - Adjust boundaries to enforce >= 60s gap:
         - train_end = boundary_70 (round down to nearest glitch time, then subtract 60s for gap)
         - val_start = first glitch time > train_end + 60s
         - val_end = boundary_85 (round down, subtract 60s)
         - test_start = first glitch time > val_end + 60s
       - Assign: train = event_time <= train_end
       - Assign: val = val_start <= event_time <= val_end
       - Assign: test = event_time >= test_start
       - Samples falling in the 60s gaps are EXCLUDED (document how many)
    4. Verify actual split ratios are within [60/10/10, 80/20/20] range
    5. Check per-class representation in each split:
       - For each class, count samples in train, val, test
       - Flag classes with 0 samples in ANY split
       - If a class has 0 test samples, attempt boundary adjustment:
         a. Shift val_end/test_start boundary to capture at least 1 sample of the missing class
         b. If still impossible (all samples of that class are in one time window), document as "not independently testable"

    **Step 2: Create split manifest CSVs**

    For each split (train, val, test), save a manifest CSV containing:
    - gravityspy_id
    - event_time (GPS)
    - ifo (H1 or L1)
    - ml_label (class)
    - ml_confidence
    - snr
    - image_path_0.5s, image_path_1.0s, image_path_2.0s, image_path_4.0s (paths to resized spectrograms from Plan 01)
    - split (train/val/test)

    **Step 3: Programmatic split verification**

    Write `scripts/05_verify_split.py` to run ALL of these checks (this is the decisive verification):

    1. **Temporal gap check**: For every test sample, compute min |t_test - t_train| across ALL train samples. Assert all >= 60s. Repeat for val vs train.
       Implementation note: Sort both arrays, use binary search for O(N log N) instead of O(N^2).

    2. **No ID overlap**: Assert intersection of gravityspy_id sets across splits is empty.

    3. **No GPS time range overlap**: Assert max(train_time) + 60 < min(val_time) and max(val_time) + 60 < min(test_time).

    4. **Class coverage**: Count classes per split. Assert all 23 classes have >= 1 sample in test.

    5. **Split ratio check**: Assert train_fraction in [0.60, 0.80], val_fraction in [0.08, 0.25], test_fraction in [0.08, 0.25].

    6. Print PASS/FAIL for each check with supporting numbers.
