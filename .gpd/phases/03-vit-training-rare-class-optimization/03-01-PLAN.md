  <action>
    Create the ViT-B/16 model builder and adapt the CNN training loop for ViT-specific requirements.

    **1. Model builder (src/models/vit_classifier.py):**
    - Load pretrained ViT-B/16 via `timm.create_model("vit_base_patch16_224.augreg_in21k_ft_in1k", pretrained=True, num_classes=23)`
    - If `augreg_in21k_ft_in1k` is not available in the installed timm version, fall back to `vit_base_patch16_224.orig_in21k_ft_in1k`, then `vit_base_patch16_224` with pretrained=True. Log which checkpoint was actually loaded.
    - Implement `get_layer_wise_lr_groups(model, base_lr, decay=0.75)`:
      - ViT-B has 12 transformer blocks (model.blocks[0] through model.blocks[11])
      - Block i gets lr = base_lr * decay^(12 - i). So block 0 (earliest) gets base_lr * 0.75^12 ~ 0.032 * base_lr, block 11 (latest) gets base_lr * 0.75^1 = 0.75 * base_lr.
      - Classification head (model.head) gets base_lr * 1.0 (no decay).
      - Patch embedding (model.patch_embed) and positional embedding get the lowest LR (same as block 0).
      - Return list of dicts for optimizer: [{"params": [...], "lr": lr_for_group}, ...]
    - First try to use timm's built-in `timm.optim.param_groups_layer_decay()` if available. If not, implement manually using named parameters.
    - Add forward pass shape assertion: input (B, 3, 224, 224) -> output (B, 23).
    - Print model summary: total params, trainable params.

    **2. Training script adaptation (src/training/train_vit.py):**
    - Copy structure from src/training/train_cnn.py but with these ViT-specific modifications:
      - Use `get_layer_wise_lr_groups()` for optimizer param groups instead of flat params
      - Base LR = 1e-4 (10x lower than CNN's 1e-3)
      - Layer decay = 0.75
      - All other hyperparameters IDENTICAL to cnn_baseline.yaml: AdamW, weight_decay=0.01, cosine warmup (5 epochs), gradient_clip=1.0, fp16, batch_size=64, early_stopping patience=10 on val_macro_f1, label_smoothing=0.1
    - Reuse DIRECTLY from Phase 2 (import, do not copy):
      - `src/data/dataset.py` (GravitySpyDataset, get_sampler)
      - `src/data/transforms.py` (get_train_transforms, get_eval_transforms)
      - `src/losses/focal_loss.py` (FocalLoss)
    - Save best checkpoint as dict with keys: model_state_dict, epoch, val_macro_f1, config
    - Log per-epoch: train_loss, val_loss, val_macro_f1, lr (for learning rate tracking)
    - Save training_log.json and training_curves.png (loss + macro-F1 per epoch)

    **3. Config (configs/vit_rare_class.yaml):**

    - Change ONLY: model.architecture -> "vit_base_patch16_224.augreg_in21k_ft_in1k", model.pretrained -> "augreg_in21k_ft_in1k", training.learning_rate -> 1e-4, add training.layer_decay -> 0.75
    - Change output_dir -> results/03-vit-rare-class, checkpoint_dir -> checkpoints/03-vit-rare-class

    - Run `python scripts/11_train_vit.py` (or equivalent entry point)

    - Monitor for red flags during training:

      - Training time > 8 hours -> check that fp16 autocast is active

    **Convergence monitoring:**
    - Training loss must decrease after warmup (epochs 6+)
    - Val macro-F1 must exceed 0.60 within 15 epochs (CNN reached 0.66 on val)
    - If val macro-F1 plateaus below 0.55 for 10 epochs, STOP and diagnose (likely LR issue or checkpoint loading failure)
